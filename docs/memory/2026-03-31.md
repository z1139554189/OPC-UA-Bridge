  - 启动时只创建空订阅通道（秒级完成）
  - 节点首次请求时按需 subscribe_data_change，值推送后写入缓存
  - 后续请求直接从缓存命中，无网络延迟
- **Session 限制**：服务器最大 Session 数很少（约 2-3 个），频繁重启会耗尽 Session，需等 60-120 秒自动释放
