## 六、关键机制

**SQLite 历史库**：WAL + 批量 buffer（100 条或 5 秒 flush），分表（按 node_id），7 天自动清理

**调度器**：60 秒间隔 → 直读 SQLite → 追加 Excel。启动时 warmup_subscriptions() 预热。Excel 占用静默跳过，失败重试 3 次 + alert.log 告警

**报表格式**：时间戳精确到秒，数值保留 2 位小数，节点显示去掉 `ns=1;s=` 前缀

## 八、经验教训

| 问题 | 解决方案 |
|------|----------|
| 服务器不支持 Read | 用 Subscription 订阅模式 |
| 历史数据稀疏 | 60 秒定期轮询强制写缓存 |
| append 清空合并单元格报错 | 首次新建 + 后续只追加新行 |
| 新节点报表无数据 | warmup_subscriptions() 启动预热 |
| Excel 被占用 | _is_file_locked() 静默跳过 |
| asyncua Python 3.14 不兼容 | 安装 Python 3.13 |

**工作流程**：遇错先查 Issues → 大操作先评估 → 交付前验证核心功能
