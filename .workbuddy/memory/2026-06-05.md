## 15:20 - 电机健康度算法 V2 重构

### 用户反馈
- 基线不应硬编码 22A，实际电流16A但可能不准确，应从运行数据自学习
- 应通过细微变化提前捕捉异常：电流波动幅度、波动频次等

### V2 算法设计（基于权威标准）
依据: ISO 20958:2013 / MCSA综述(IEEE2023) / CUSUM-EWMA漂移检测(TU Wien 2024) / NEMA MG1

#### 新 5 维度（替换旧的5维度）

| 维度 | 权重 | 方法 | 依据 |
|------|------|------|------|
| CUSUM 累积漂移 | 30% | 双侧累积和, k=0.5σ, h=5σ | ISO 20958, Montgomery SPC |
| 短期波动增幅 | 25% | 滚动窗口σ vs 基线σ | SPC 统计过程控制 |
| EWMA 趋势偏移 | 20% | λ=0.15 指数加权移动平均 | TU Wien 2024 (平均漂移最优) |
| 异常偏移频次 | 15% | ±2σ正常带偏移频率比 | 统计异常检测 |
| 峭度变化 | 10% | 超值峰度 Δ(Excess Kurtosis) | 轴承冲击故障特征 |

#### 自适应基线策略
1. 优先用历史基线期（>30天前）数据的中位数 → 抗离群值
2. 基线期无数据时，将30天数据拆分为前/后半 → 前半做基线，后半做CUSUM分析
3. 数据覆盖 < 3天 → CUSUM 分数下限 40（防止误报）
4. 铭牌值 22A 仅作参考线，不参与基线计算

#### 删除的旧指标
- 三相不平衡度（无真实三相数据）
- 启动电流特征（无法区分启动峰值和过载峰值）
- 峰值合规率（无负载上下文）

### 前端更新
- 维度标签更新为 V2 指标名 + 详细参数
- 当前电流显示 EWMA 偏移 σ 数（而非旧 drift_pct）
- 趋势图新增 ±2σ 正常带（绿色虚线区域）
- 基线来源标注（自适应统计中位数/数据拆分/数据不足）
- 数据不足时显示 info 级别提示

### 当前数据状况
- 自适应基线 = 15.95A（从数据自学习，非 22A）
- 数据仅覆盖 1 天（3300+ 点），data_sufficient=false
- CUSUM 因数据不足被限制最低分 40，避免误报
- 随数据积累（≥7天），各维度评分将逐步精确

### 改进 3: 威布尔分布替换线性外推
- 旧方案：`剩余天数 = (1.3×基线 - 当前值) / 每天退化速率`（无依据）
- 新方案：双参数威布尔 R(t)=exp(-(t/η)^β), β=3.5（磨损失效）, R_target=0.90
  - η 由 EWMA 退化速率动态校准
  - 失效阈值改为基线+3σ（旧为 1.3×基线，无统计依据）
- 数据保护：`data_span_days < 7` 时不输出寿命估计（返回 365），避免 1 天数据给出"28天"的错误判断

### 关键技术决策
- 停机阈值用自动分界法而非硬编码百分比——用户明确要求权威依据
- baseline_days 默认 90 天（Query 参数，可调 30~180）
- OPC UA 实时读取失败静默回退，不中断 API 响应
