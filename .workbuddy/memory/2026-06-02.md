## 09:24 - 看板轮询间隔优化

### 背景
用户询问卡片实时值刷新间隔，原为 3 秒。

### 改动历程

| 时间 | 改动 | 说明 |
|------|------|------|
| 09:24 | 3s → 1s | 简单改 interval |
| 09:32 | 1s → 0.5s | 用户要求进一步加速 |
| 09:40 | 分析 0.5s 风险 | 指出 setInterval 可能造成请求重叠（雪崩） |
| 10:04 | 改 setTimeout 递归 | 防重叠：poll 跑完再等 500ms 才启动下一次 |

### 关键发现
- OPC UA Bridge 使用订阅模式，`create_subscription(500)` → 服务器每 500ms 推送一次
- 数据链路延迟：OPC 500ms + 看板 500ms ≈ 最大 1 秒

### 代码改动（dashboard.html）

**旧**：
```javascript
setInterval(pollRealtime, 1000);
```

**新**：
```javascript
// 防重叠轮询：poll 跑完再等 500ms 才启动下一次
let _pollTimer = null;
async function startPollLoop() {
  try { await pollRealtime(); } catch (_) {}
  _pollTimer = setTimeout(startPollLoop, 500);
}
startPollLoop();
```
