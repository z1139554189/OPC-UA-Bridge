## 预测维护报告 — 方向判断逻辑修复

### 问题
用户发现偏度和自相关的好坏方向判断有误：
1. 偏度 `+12.89%` 误判为"好方向"（实际上偏度从负到正，|偏度|增大 = 退化）
2. 自相关两个电机，一个标好方向一个标坏方向，逻辑不一致

### 根因
- 偏度配置为 `up_bad`（上升=坏），但偏度是有符号特征：健康≈0，退化时|偏度|增大（无论正负）
- 自相关同理：应该看|值|是否减小（相关性流失），而非简单看原始值上升/下降
- 原代码用 `effective_trend`（Sen斜率方向）判断，对自相关还受了 `slope_per_day` 符号翻转的 bug 影响

### 修复（V4.7 补丁）
1. `FEATURE_DIRECTION` 新增两种类型：
   - `abs_up_bad`: |值|增大=坏方向（偏度）
   - `abs_down_bad`: |值|减小=坏方向（自相关）
2. 方向判断逻辑（第970行附近）加特殊分支：
   - 偏度：`is_bad = (abs(last) > abs(first))`
   - 自相关：`is_bad = (abs(last) < abs(first))`
3. 同时修复了自相关 `slope_per_day` 翻转前未保存原始斜率的 bug（弱趋势方向误判）

### 验证
- 偏度 `-0.2556 → 0.4777`：|值|增大 → ⚠ 坏方向 ✅
- 自相关 `-0.0207 → 0.0151`：|值|减小 → ⚠ 坏方向 ✅
