## 15:12 - 修复 Bridge 连接失败（两个 bug）

### Bug 1: 心跳崩溃 → 退避链
- client.py:565 `self._yielded` 不存在 → AttributeError
- 修复：改为 `self.is_yielded()`
- 影响：心跳 loop 崩溃后触发退避，每次重启退避时间指数增长

### Bug 2: 启动时 _collect_nodes 为空
- 启动流程：`start()` → `_collect_loop()` → `_connect_opc()` 订阅 `_collect_nodes`
- 但 `_collect_nodes` 为 `set()` 空集合 → 订阅 0 节点 → 15 秒无推送 → 退避 499 秒
- 修复：在 `main.py` lifespan 中，`start()` 之前预注册 20 个默认节点
- `opcua_client.add_nodes([20 个节点])` 必须在 `opcua_client.start()` 之前调用

## 15:48 - 修复历史查询 Canvas 上下文冲突
- **错误**：`Canvas with ID 'historyChart' has an existing context`
- **根因**：Chart.js `destroy()` 不彻底清除 canvas 的 2D 上下文，再次 `getContext('2d')` 报错
- **修复**：`drawChart()` 中 destroy 后移除旧 canvas，`createElement('canvas')` 重建，再 `getContext`
- **教训**：Chart.js 重复查询必须走"销毁 → 移除 DOM → 重建 canvas → getContext"流程

## 15:51 - 修复 Chart.js 日期适配器缺失
- **错误**：`Uncaught ReferenceError: dateFns is not defined`
- **根因**：Chart.js v4 时间轴 (`type: 'time'`) 需要日期适配器，只引入 chart.js 不够
- **修复**：在 chart.js 之前引入 `date-fns` + `chartjs-adapter-date-fns` 两个 CDN
- **引入顺序**：date-fns → chartjs-adapter-date-fns → chart.js（顺序不能错）

## 16:00 - 修复导出 Excel 时间对齐和空白单元格问题
- **现象1**：时间间隔不是每分钟，FIT 和 FIQ 各自成行，每一行都带不同毫秒时间戳
- **现象2**：大量空白单元格，FIT 行 FIQ 为空，FIQ 行 FIT 为空
- **根因**：导出代码用精确字符串 `p["timestamp"] == ts` 匹配来对齐节点，不同节点的毫秒时间戳不可能相等
- **修复**：改为固定时间桶对齐方案
  - 从 `start_time` 开始按 `interval_seconds` 生成固定桶序列
  - 每个桶取该时间段内最新一条值
  - 每行时间戳格式化为 `YYYY-MM-DD HH:MM:SS`（去掉毫秒）
  - 所有节点在同一行的数据来自同一时间桶
- **文件**：`src/api/main.py` `batch_history_export` 函数
