## 新增工作流程规则

### 文件清理流程
1. **全面扫描**：扫描整个目录树，不遗漏任何角落
2. **分析分类**：区分核心文件 vs 无用文件
3. **列表确认**：列出待删除项+原因，请用户确认
4. **执行删除**：删除确认的无用文件
5. **更新记忆**：记录到 MEMORY.md

### Git 推送流程
1. `git status` 确认改动
2. `git ls-files` 确认无遗漏
3. `git add -A` 添加所有变更
4. `git commit -m "..."` 提交
5. `git push origin main` 推送

**推送前必查**：确保一个不落，特别是类似 server.js 这种从未进过 git 的文件
