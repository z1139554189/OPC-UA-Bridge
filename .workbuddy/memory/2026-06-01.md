## 10:12 - 位号选择器重写（WPS 筛选风格）

### 问题
- 全选按钮无效：`document.querySelector('#lbl_FIT_05R301F01.PV input')` 中 `.` 被解析为 CSS 类选择器，导致匹配失败
- 勾选框无视觉反馈
- 用户要求改成 WPS 筛选风格

### 解决方案
- **改用 `data-node` 属性**替代含 `.` 的 id，用 `[data-node="FIT...PV"]` 规避 CSS 歧义 ← 核心 Bug fix
- 新增搜索框（模糊匹配位号），不匹配项隐藏，可见数量实时显示
- 顶部操作栏：全选 / 清空 / 选中搜索结果

### 关键教训
label 的 id/querySelector 里不能含 CSS 特殊字符（`.`、`[`、`:`等），统一改用 data-* 属性是最干净的方案
